# Internacionalización con next-intl

> INVESTIGADO: next-intl docs oficiales (next-intl.dev) — v4 vigente.

Gestión de idiomas con next-intl en Next.js App Router. Soporte multi-idioma con routing automático.

## Instalación y Configuración

```bash
npm install next-intl
```

Archivos base:

```json
// messages/es.json
{
  "HomePage": {
    "title": "Productos Destacados",
    "description": "Explora nuestra colección"
  },
  "auth": {
    "login": "Iniciar Sesión",
    "register": "Registrarse"
  }
}

// messages/en.json
{
  "HomePage": {
    "title": "Featured Products",
    "description": "Explore our collection"
  },
  "auth": {
    "login": "Log In",
    "register": "Register"
  }
}
```

## Configuración con App Router

### 1. Archivo de configuración de idioma

```ts
// i18n/request.ts
import { getRequestConfig } from 'next-intl/server';

export default getRequestConfig(async ({ requestLocale }) => {
  // El locale se resuelve automáticamente desde la URL
  const locale = await requestLocale;
  const messages = (await import(`../messages/${locale}.json`)).default;
  return { locale, messages };
});
```

### 2. Layout raíz con provider

```tsx
// app/[locale]/layout.tsx
import { NextIntlClientProvider } from 'next-intl';
import { getMessages } from 'next-intl/server';

export default async function LocaleLayout({
  children,
  params,
}: {
  children: React.ReactNode;
  params: Promise<{ locale: string }>;
}) {
  const { locale } = await params;
  const messages = await getMessages();

  return (
    <html lang={locale}>
      <body>
        <NextIntlClientProvider messages={messages}>
          {children}
        </NextIntlClientProvider>
      </body>
    </html>
  );
}
```

### 3. (Opcional) Routing con middleware

Si necesitas routing basado en locale (ej. `/es/producto` y `/en/product`):

```ts
// middleware.ts
import createMiddleware from 'next-intl/middleware';

export default createMiddleware({
  locales: ['es', 'en'],
  defaultLocale: 'es',
  localeDetection: true,
  localePrefix: 'always',
});

export const config = {
  matcher: ['/((?!api|_next|_vercel|.*\\..*).*)'],
};
```

## Uso en Server Components

```tsx
import { getTranslations } from 'next-intl/server';

export default async function HomePage() {
  const t = await getTranslations('HomePage');
  return <h1>{t('title')}</h1>;
}
```

## Uso en Client Components

```tsx
'use client';
import { useTranslations } from 'next-intl';

export function LoginButton() {
  const t = useTranslations('auth');
  return <button>{t('login')}</button>;
}
```

## TypeScript: Autocompletado

```ts
// next-intl.d.ts
type Messages = typeof import('./messages/es.json');
declare interface IntlMessages extends Messages {}
```

Esto habilita autocompletado y type-checking en `t('...')`.

## Rutas por Idioma (con middleware)

| Ruta | Idioma |
|------|--------|
| `/es/producto/123` | Español |
| `/en/product/123` | Inglés |

Los links internos deben usar navegación relativa al locale:

```tsx
import { Link } from '@/i18n/routing';

<Link href="/producto/123">Ver producto</Link>
```

## Detección de Idioma del Navegador

`localeDetection: true` en el middleware lee el header `Accept-Language`. Si el usuario tiene `es-MX,es;q=0.9,en;q=0.8`, redirige a `/es`. Si la detección falla, usa `defaultLocale: 'es'`.

## Funcionalidades adicionales de next-intl v4

- **Formato de números:** `useFormatter()` para números, monedas, porcentajes
- **Formato de fechas:** `useFormatter().dateTime()` para fechas localizadas
- **Mensajes con HTML:** `<TranslatedHtml>` o `t.rich()` para markup en traducciones
- **Plurales:** `t('items', { count: 5 })` con reglas `{ count: 0, one: '...', other: '...' }`
- **Cardinales y ordinales:** `t.ordinal('position', { n: 3 })` → "3°" o "3rd"
